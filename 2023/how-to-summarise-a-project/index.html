如何描述一段科研经历

——以准备研究生院申请为例

我自己经历了两个研究生院申请季，交了总共交了十几封（英文）申请信，再加上这半年前前后后帮几个同学改了申请文书，所以想趁此机会回顾一下我自己写申请文书的经验。我是工科生，接触过的也都是理工科项目的申请，而对于理工科同学来说，申请信里篇幅最大的部分就是描述以往的科研经历，所以本文希望集中讨论这一点——如何在申请材料中描述一段科研经历。

太长不看 #

总结起来就只有一个要点：know your shit

一个项目的结构 #

简单地描述一个科研项目，就像写文章摘要，要五脏俱全地写出一整个项目的来龙去脉：

  1. 背景知识——所有科研文章开头都会提两句给外行看的大废话，说说这个项目涉及到了什么大方向上的问题，为什么这个大方向是值得研究的。比如说：地球人口越来越多，全球能耗越来越大，所以我们想要开发低功耗的电灯泡。再比如说：II 型糖尿病患者越来越多，现在的胰岛素泵太弱智了不能实时反馈，造成了很多麻烦。这两句大废话是一定要写的，这样才能迅速让读者知道你在做什么，对接下来的内容有所预期。
  2. 问题陈述——然后就该详细讲讲你要为这个背景里的大问题提供什么方向上的小小进步，换句话说，在这个项目里，你想要解决一个小问题，而这个小问题能够帮助解决这个大问题。为什么这个小问题很重要呢？它们之间的具体关联又是什么样的？——这是这部分的要义。
  3. 实验——用了什么东西、做了什么测试、分析计算用了什么方法
  4. 结果——总得有点结果！失败的结果也是结果，因为申请信不是发论文，所以可以更诚实地描述“没有结果”。这实际上是我对学术论文不满的一点，很少有人会在文章里说怎样行不通，然而行不通才是科研工作中更常见的部分。
  5. 讨论结论和重要性——从这些结果里能够得出什么结论？不能得出什么结论？这个小项目是不是完成了第二步中预期的小问题呢？如果解答了这个小问题，那就是一个很好的成果，对这个大问题和大问题将要解答的大意义都有贡献，请大夸特夸！如果不能得出决定性结论，那也是有意义的工作，可能说明了这个小问题并不能解决这个大问题，这个大问题不应该拆分出这个小问题。

这是描述一个项目的最基本顺序，我的建议是不要觉得死板公式化，先按照模板顺序走一遍。这既然是所有人都熟悉的模板，读者读起来就会很轻松，因为完全知道接下来应该是什么，很容易跟上你的叙述。

作为一个本科生或者硕士生，你可能觉得自己的研究工作是非常渺小的一部分，科学大问题你根本碰不着，但是你的工作确实是大问题的一部分，那就一定要写上背景知识和重要性，说明你知道自己在做什么，为什么要做这个。最简单的办法是，老师和师兄师姐怎么给你画大饼，你就照着描一遍给录取委员会看。虽然大饼很大，但是都不是你做的，所以提完这两句背景知识，重要的部分还是你做了什么。

你的贡献 #

你在这个项目里具体做了什么贡献呢？以下几个简单的问题可以帮助你组织这一部分：

  • 遇到了什么问题？
  • 想了什么办法来解决？为什么用这个办法？
  • 结果怎么样？定性定量地来说，你的办法贡献了多少？

显然你的贡献主要囊括了上一节说到的3-5点，所以这也应该是整段的核心内容。上一节也提到了，你的工作可能不是很成功，没有太好的结果。尽管如此，还是应该把整个过程描述出来，包括你尝试了什么方法、不如预期的结果是什么结果etc. 这些都能显示你的工作量，如果什么也不写，就等于什么也没做。

措辞注意事项 #

从语言风格上来说，写申请信和写论文不一样，申请信需要突出申请人本人，所以可以写很多“我”起头的句子，不用担心看起来ego很大，全是我我我。对于英文写作并不非常熟练的同学，我还建议少用长句子，少写各类从句，几乎不要用逗号。特别是实验过程，可以试着先用短句从头到尾写一遍，完全按照时间顺序，先做了什么又做了什么。在第二遍阅读修改的时候再替换一点高级句法。就算不改，就全写简单短句子也完全不要紧，能看懂的段落永远比主谓宾都找不齐的三行长句要好。

虽然句法不应该追求华丽，但是在实验内容上，请不要害怕用非常技术细节的词汇，因为作为要申请研究生院的同学，你应该有相当的科学词汇量。哪怕是在申请信里，也应该遵守基本的科学写作规范，比如说第一次提到的东西要写全称，然后在括号里标注缩写，而不是一开始就用缩写，因为读者不一定知道你在缩写什么。在提到一些不是很常见的概念时，也应该简单解释一下。

读者期待看到什么 #

申请信的读者当然是可能会招你的录取委员会老师，他们想看到的是一个有能力进入研究生院学习工作的同学。那么首先他们想看到你的背景知识和本系的研究方向多少有点关系，然后你有基础的研究工作训练，比如说能理解实验目的，有能力分析实验结果，有想法来安排下一步的工作。这些都是基本的qualification, 只能保证你及格，如果申请竞争很激烈，我也不会写更高级的作文，那就另请高明吧！

在此基础上准备简历和面试 #

那么简历和面试的内容都是一样的，只是因为形式不同，需要一些删改。只要记住这个结构顺序，突出重点——也就是你的贡献，就好了。面试的特别之处就是老师可以提问，可以问更多更细节的内容，那么你也应该有所准备。一个简单的练习方法是对同一个项目准备一个三分钟的讲稿，再准备一个十分钟的讲稿。

就是这些。祝同学们学业顺利！
